What happens when you initialize a repository? Why do you need to do it?

A: The respository is tracked by Git. It creates a new .git config file under the working directory.

How is the staging area different from the working directory and the repository? What value do you think it offers?

A: Working area has all files that are added to the git repository. Staging area has files that are yet to be added to the repository. Staging area lets you decide which files need to be tracked on git repository.

How can you use the staging area to make sure you have one commit per logical change?

A: Using 'git add' command we can stage files that can be grouped logically.

What are some situations when branches would be helpful in keeping your history organized? How would branches help?

A: Branches would really help if you pick an experimental approach where you want to try a different path or when you want to add a feature that diverts from the original concept.
